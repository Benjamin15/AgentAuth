# Quality Gates Enforcement for AI Agents

To ensure that every code change maintains the project's high standards for quality, stability, and security, all AI coding assistants (including myself) MUST follow these mandatory quality gates.

## Mandatory Quality Check Workflow

Before concluding any task, submitting a PR, or presenting a walkthrough, YOU MUST execute the following steps without exception:

1.  **Format and Lint**: Run the automated quality checks to ensure adherence to PEP 8 and project-specific styling.
    -   Command: `ruff check agentauth --fix && ruff format agentauth`
2.  **Static Type Check**: Ensure 100% type safety and resolve any `mypy` errors.
    -   Command: `mypy agentauth`
3.  **Run Tests**: Execute the full test suite and ensure ALL tests pass.
    -   Command: `pytest --cov=agentauth --cov-report=term-missing`

> [!IMPORTANT]
> A task is ONLY considered complete when all of the above commands exit with a successful status (exit code 0). If any check fails, YOU MUST fix the underlying issues and re-run all quality gates.

## Slash Command Shorthand
You can run all of the above steps in a single command using the workspace workflow:
-   Command: `/run-quality-gates`

## PR Quality Checklist
Every PR description generated by an AI assistant must include a "Quality Verification" section confirming that these steps were completed.
